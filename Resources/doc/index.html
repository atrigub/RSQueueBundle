Simple, fast and agile redis-based queue system
for Symfony2 developers

The project has been created completely open source. You are completely free to modify the code and share it with the community.

The code is documented following the Symfony2 coding standard completely strict. We like the clean and structured code.

The project is based on the well-known redis key-value system. It is a wonder! Uses the bundle snc/redis-bundle for Symfony2.

Installation

Compatible with Symfony 2.3 and later

Install redis

First of all, you must install redis service in your system. Otherwise, this bundle will not work. You can find a very intuitive and elaborated guide here.


    wget http://download.redis.io/redis-stable.tar.gz
    tar xvzf redis-stable.tar.gz
    cd redis-stable
    make

Install RSQueueBundle

Once you have redis installed properly, you need to add a require line with RSQueueBundle


    "require": {
        "php": ">=5.3.3",
        "symfony/symfony": "2.3.*",
        ...
        "mmoreram/rsqueue-bundle": "dev-master"
    },

Then you need to use composer to update your project dependences


    php composer.phar update

And register the bundle in your appKernel


    return array(
        // ...
        new Mmoreram\RSQueueBundle\RSQueueBundle(),
        // ...
    );

Et voilà! You can now start using the power of queues in your project.

Serializers

Cuando guardamos nuestros datos en una cola para que un proceso externo tenga acceso a ellos, debemos tener muy claro en que formato vamos a guardarlo para que los entienda y pueda trabajar con ellos. Para ello, este bundle implementa dos formas básicas de serialización y la posibilidad de añadir nuevas, simplemente implementando una interface

Los serializers que implementa son el serializador nativo de php ( serialize, unserialize ) y el serializador de json ( json_encode, json_decode )

Para configurar que serializador utilizar debemos añadir la opción serializer. Este parámetro acepta los valores PHP, JSON y un namespace de una clase existente en el caso de que queramos utilizar un serializer propio. En caso que sea así, tal clase deberá implementar la interface Mmoreram\RSQueueBundle\Serializer\Interfaces\SerializerInterface 


    rs_queue:
        serializer: ~

Por defecto, el valor de este parámetro es JSON
